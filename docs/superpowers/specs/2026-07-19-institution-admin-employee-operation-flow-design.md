# 차량운행일지 기관관리자·기관직원 운영 흐름도 설계

## 목적

기존 `vehicle-drive-log-architecture.png`가 개발·기획 관점의 기술 구조를 설명하는 것과 달리, 새 이미지는 기관에서 서비스를 도입하고 배포하며 일상적으로 사용하는 과정을 사람의 관점에서 설명한다.

독자가 자신의 역할을 먼저 찾고 다음 행동과 사용할 수 있는 기능을 빠르게 이해하도록 기관관리자와 기관직원을 두 명의 주인공으로 표현한다. 시스템 관리자와 기술 플랫폼은 주 흐름을 방해하지 않는 지원 영역으로 배치한다.

## 대상 독자

- 차량운행일지 도입을 검토하거나 신청하는 기관관리자
- 기관의 차량, 직원, 예약, 운행기록과 보고 업무를 담당하는 관리자
- 초대를 받아 차량을 예약하고 운행일지를 작성하는 기관직원

## 결과물

- 파일: `docs/images/vehicle-drive-log-organization-journey.png`
- 형식: 한글 가로형 고해상도 PNG
- 권장 크기: 3200×1800px 또는 이에 준하는 16:9 비율
- 용도: README, 서비스 소개자료, 도입 안내 및 내부 사용자 교육
- 기존 종합 구조도와 색상·선·모서리 스타일을 맞춘 짝 이미지

## 핵심 설계 결정

표나 기능 매트릭스가 아니라 두 역할의 여정을 따라가는 프로세스 흐름도로 제작한다.

1. 기관관리자와 기관직원 영역을 크고 선명한 두 개의 주 흐름으로 표시한다.
2. 각 역할 영역 왼쪽에 인물 아이콘과 역할 설명을 배치한다.
3. 역할별 업무를 왼쪽에서 오른쪽으로 연결된 카드와 화살표로 나타낸다.
4. 기관관리자의 직원 초대가 기관직원의 가입으로 이어지는 인계선을 표시한다.
5. 기관직원이 작성한 예약·운행 데이터가 기관관리자의 현황 관리로 돌아가는 환류선을 표시한다.
6. 시스템 운영과 공통 기술은 상단의 얇은 지원 영역으로 축소한다.
7. 기능과 외부 도구는 각 업무 카드 안의 짧은 태그와 하단 도구 영역으로 표현한다.

## 정보 구조

### 제목 영역

- 제목: `차량운행일지 — 기관 도입부터 직원 사용까지`
- 부제: `기관관리자와 기관직원, 두 사람의 관점으로 보는 업무 흐름`

### 서비스 지원 영역

주 흐름 상단에 `서비스가 뒤에서 안전하게 지원합니다`라는 얇은 띠를 둔다.

- 기관 심사·개설
- 인증과 기관별 권한 보호
- 알림과 캘린더 동기화
- 서비스 상태 관제와 오류 대응
- 백업과 운영 지원

시스템 관리자는 독립적인 주인공으로 표현하지 않고 위 지원 업무 안에 포함한다.

### 기관관리자 관점

역할 설명은 `우리 기관에 도입하고 직원과 차량을 관리합니다`로 표시한다.

1. **서비스 도입 신청**
   - 기관 정보와 증빙자료 제출
   - 기능 태그: 신청서, 증빙 업로드
2. **승인·온보딩**
   - 승인 결과 확인과 기관 운영 시작
   - 기능 태그: 이메일, 알림톡, 온보딩
3. **차량·운영 설정**
   - 차량, 결재선, 기관별 기능 설정
   - 기능 태그: 차량 관리, 결재선, 기능 설정
4. **직원 초대·배포**
   - 직원 등록과 앱 사용 안내
   - 기능 태그: 초대 코드, 직원 관리, 공지
5. **운영 현황 관리**
   - 예약과 운행일지 확인 및 승인
   - 기능 태그: 예약 승인, 운행일지 관리, 알림
6. **보고·차량 개선**
   - 통계와 차량 유지관리 데이터를 기관 운영에 활용
   - 기능 태그: 통계·월간보고, PDF·Excel, 정비·주유·하이패스

### 기관직원 관점

역할 설명은 `차량을 예약·운행하고 간편하게 기록합니다`로 표시한다.

1. **초대받고 가입**
   - 초대 코드로 기관 참여
   - 기능 태그: Google 로그인, 초대 코드
2. **앱 설치·연동**
   - 홈 화면 설치와 알림·일정 연결
   - 기능 태그: PWA, 푸시 알림, Google Calendar
3. **차량 예약**
   - 필요한 차량과 시간 선택
   - 기능 태그: 예약 달력, 반복 예약, Slack
4. **운행·일지 작성**
   - 빠른 운행과 OCR을 이용한 현장 기록
   - 기능 태그: 빠른 운행, OCR, TMAP, 오프라인 동기화
5. **내 기록 확인**
   - 개인 운행 결과와 차량 이력 확인
   - 기능 태그: 내 기록, 차량 이력, 즐겨찾기

### 역할 간 인계와 환류

- 기관관리자의 `직원 초대·배포`에서 기관직원의 `초대받고 가입`으로 주황색 점선 화살표를 연결한다.
- 기관직원의 `운행·일지 작성`과 `내 기록 확인`에서 기관관리자의 `운영 현황 관리`로 주황색 점선 환류선을 연결한다.
- 인계선에는 `직원 초대·사용 권한`, 환류선에는 `운행 기록·현황`을 표시한다.

### 관점별 핵심 기능·도구

하단에 두 역할의 대표 도구를 짧게 요약한다.

- 기관관리자용: 직원·차량 관리, 예약·일지 관리, 통계·보고서, 정비·주유·하이패스
- 기관직원용: PWA, 차량 예약, 빠른 운행·OCR, TMAP·Google Calendar, 내 기록

## 시각 디자인

- 기존 종합 구조도와 어울리는 밝은 배경, 높은 명암 대비, 둥근 카드 스타일을 사용한다.
- 기관관리자는 파랑, 기관직원은 청록, 서비스 지원은 보라, 역할 간 인계는 주황으로 구분한다.
- 색상만으로 역할을 구분하지 않고 인물 아이콘, 역할 제목, 영역 경계와 라벨을 함께 사용한다.
- 동일 크기의 격자 셀이나 세로 단계선을 사용하지 않는다.
- 빈 단계를 만들지 않고 각 역할에 필요한 업무 카드만 배치한다.
- 업무 카드는 큰 제목, 한 줄 설명, 2~4개의 기능 태그 순서로 읽힌다.
- 화살표가 카드와 글자를 가리지 않도록 흐름 영역의 여백을 확보한다.
- 작은 화면에서도 역할 제목과 업무 단계가 먼저 읽히고, 확대하면 세부 기능 태그를 확인할 수 있게 한다.

## 정확성 기준

- 실제 구현된 기능과 연동만 사용한다.
- 기관 신청, 기관 승인, 초대 코드 가입, 차량 예약, 운행일지 OCR, 빠른 운행, 통계·월간보고, PDF·Excel, 정비·주유·하이패스, PWA, Google Calendar, TMAP, Slack을 코드와 README 기준으로 검증한다.
- `기관관리자`, `기관직원`, `운행일지`, `하이패스` 등 한글 표기를 일관되게 유지한다.
- 지원 기술은 사용자 여정을 설명하는 데 필요한 범위만 표시하고 Firebase 내부 구성이나 Cloud Functions 유형은 반복하지 않는다.

## PNG 제작과 검증

1. 승인된 두 관점 시안을 기준으로 가로형 이미지를 제작한다.
2. 결과물을 `docs/images/vehicle-drive-log-organization-journey.png`에 저장한다.
3. PNG를 원본 크기로 열어 한글 깨짐, 잘림, 카드 겹침, 화살표 방향을 확인한다.
4. 축소 화면에서 기관관리자와 기관직원 두 관점이 즉시 구분되는지 확인한다.
5. 각 역할의 업무 순서와 기능 태그가 설계 문서와 일치하는지 대조한다.
6. 기존 `vehicle-drive-log-architecture.png`와 함께 놓았을 때 제목, 색상과 시각적 무게가 조화를 이루는지 확인한다.
