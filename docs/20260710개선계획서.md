# 2026-07-10 품질 개선 실행 계획

> **에이전트 작업자용:** 구현 시 `superpowers:subagent-driven-development`(권장) 또는 `superpowers:executing-plans`를 사용해 아래 작업을 순서대로 수행한다. 모든 실행 단계는 체크박스로 추적한다.

**목표:** 2026-07-10 실측 평가 81/100에서 확인된 보안 배포 미완료, 인증 E2E 비결정성, 낮은 프론트 커버리지, 약한 접근성 검사, 부족한 번들 여유, 문서 드리프트를 4주 안에 해소해 재평가 88점 이상을 달성한다.

**아키텍처:** 신규 기능을 추가하지 않고 현재 구조를 유지한다. 변경은 보안 배포, 테스트 기반 강화, 단계적 App Check 적용, 품질 게이트 강화, 문서 정합성의 다섯 트랙으로 분리하며 각 트랙은 독립 커밋·독립 검증·독립 롤백이 가능해야 한다.

**기술 스택:** Node.js 22, React 19, TypeScript 5.9, Vite 7, Vitest 4, Playwright 1.59, Firebase Auth/Firestore/Functions/Hosting, Zod 4, GitHub Actions

## 전역 제약

- 모든 검증과 빌드는 Node.js 22.x에서 실행한다.
- 프로덕션 배포는 `master` 푸시 후 GitHub Actions 경로만 사용한다.
- 보안 변경, 테스트 변경, 문서 변경은 서로 다른 커밋으로 분리한다.
- 현재 작업 트리의 사용자 변경을 덮어쓰거나 되돌리지 않는다.
- 테스트는 실패 재현 → 최소 수정 → 통과 확인 순서를 지킨다.
- App Check는 한 번에 전체 적용하지 않고 세 배치로 나누며 각 배치 후 2영업일 관찰한다.
- 커버리지 하한은 실측값보다 최소 1%p 낮게 설정해 비결정적 소수점 변동을 흡수한다.
- 의존성 추가는 접근성 자동 검사에 필요한 `@axe-core/playwright` 한 개로 제한한다.

- [ ] **4단계: 마이그레이션을 dry-run/verify-only/실행 모드로 분리한다.**

`scripts/lib/googleOauthMigration.ts`에 순수 함수를 구현한다.

```ts
export type MigrationMode = 'dry-run' | 'verify-only' | 'execute';

export function parseMigrationMode(argv: string[]): MigrationMode {
  if (argv.includes('--dry-run')) return 'dry-run';
  if (argv.includes('--verify-only')) return 'verify-only';
  return 'execute';
}

export function shouldMoveOauth(oauth: unknown): boolean {
  return Boolean(
    oauth && typeof oauth === 'object' &&
    'refreshToken' in oauth &&
    typeof (oauth as { refreshToken?: unknown }).refreshToken === 'string' &&
    (oauth as { refreshToken: string }).refreshToken.length > 0,
  );
}
```

`scripts/migrateGoogleOauthToPrivate.ts`는 위 함수를 import하고 실행과 종료 코드만 담당한다.

```ts
import { parseMigrationMode, shouldMoveOauth } from './lib/googleOauthMigration';

migrate(parseMigrationMode(process.argv.slice(2))).catch((error) => {
  console.error(error);
  process.exitCode = 1;
});
```

`dry-run`은 쓰기를 수행하지 않고 `movable`만 집계한다. `verify-only`는 `users` 문서의 `googleOauth.refreshToken` 잔존 수를 계산하며 1건 이상이면 종료 코드 1을 반환한다.

- `logout(): Promise<void>`는 `signOut → clearQueue → clearOfflineCache → / 이동` 순서를 보장한다.
- `handleSubmit(event, deps): Promise<void>`는 검증 실패 시 쓰기를 수행하지 않는다.
