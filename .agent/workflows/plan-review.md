# /plan-review — 기획·스코프 리뷰 가동

이 커맨드는 [planning-scope-review 룰](../rules/planning-scope-review.md)을 **명시적으로** 발동하는 얇은 핸들이다. 모든 절차·기준은 룰 파일이 단일 원본이며, 여기서 중복 기술하지 않는다.

## 절차

1. **대상 확인**: 커맨드와 함께 기획 대상(새 기능·변경)이 주어졌으면 그것을, 없으면 사용자에게 "무엇을 기획할지" 한 줄 받는다. 대상 없이는 진행하지 않는다.

2. **룰 적용**: [planning-scope-review](../rules/planning-scope-review.md)를 읽고 순서대로 수행한다 —
   전제 도전 → 기존 코드 레버리지 → 구현 대안 2~3개(최소안+이상안) → 스코프 모드 합의 → 가드레일 점검.
   - **EXPANSION은 기본값이 아니다.** 사용자가 "크게 가자"고 명시할 때만 가동(규모 적합 축소 편향).
   - 모든 스코프 변경은 AskUserQuestion으로 명시 opt-in. 모드를 정하면 그 모드에 commit.

3. **승인**: 합의된 계획을 plan mode의 ExitPlanMode로 사용자 승인받은 뒤 구현에 들어간다.

> 코드를 작성하는 커맨드가 아니다. 구현 *전* 기획·합의 단계만 담당한다.
