# PWA & 모바일 우선 (Mobile-First) 개발 원칙

차량운행일지 플랫폼은 구조상 **운전 중 휴식 시간이나 주차장 등 모바일 환경에서 가장 많이 쓰이는 앱**입니다. 
AI 에이전트와 모든 개발자는 UI 수정 및 신규 기능 추가 시 아래 원칙을 위배하지 않아야 합니다.

## 핵심 규칙 

### 1. 모바일 뷰 우선 (Mobile Layout First)
- CSS나 Tailwind 레이아웃(flex, grid)을 짤 때 모바일 크기(예: `max-w-md`, `max-w-lg`)에서 화면이 스크롤 없이 예쁘게 수납되는지부터 고려합니다.
- 좌우 패딩을 너무 많이 잡아 모바일에서 텍스트가 잘리는 일이 없도록, 작은 화면에서는 최소 패딩(예: `p-2`, `p-4`)을 유지하세요.

### 2. 터치 최적화 (Touch Targets)
- 버튼(`button`), 링크(`a`), 입력창(`input`, `select`)의 **최소 높이는 48px**를 권장합니다.
- 패딩을 통해 충분한 클릭 공간을 확보하여 운전자가 폰 화면을 무심코 터치해도 다른 버튼이 눌리지 않게 하세요.

### 3. 화면 하단 여백 및 Floating 요소 주의
- 아이폰 사파리나 안드로이드 하단 네비게이션 바 때문에 버튼이 가려질 수 있습니다.
- `fixed bottom-0` 으로 띄우는 네비게이션/버튼이 있을 때는 안전 영역(Safe Area) 패딩(`pb-safe`, `env(safe-area-inset-bottom)`)을 잊지 마세요.

### 4. 짧은 상호작용
- 운전자가 입력해야 하는 사항(주행거리, 목적지, 메모 등)은 최대한 드롭다운이나 칩(Chip) 버튼 형태로 제공하세요. 타이핑을 최소화하는 것이 핵심입니다.
- AI 패턴 기반 예약, 한 번 클릭 예약 등이 이에 해당합니다.

### 5. PWA (오프라인 지원) 고려
- 앱이 '홈 화면에 추가'되어 네이티브 앱처럼 동작하므로, 네트워크가 불안정한 상황에서도 흰 화면 대신 캐시된 뼈대 화면이라도 뜨는지, 스피너 UI가 있는지 확인하세요.
- 사용자에게 브라우저 특유의 팝업(새 탭, 윈도우 스크롤) 대신 모달/바텀시트(Bottom Sheet) 인터페이스를 적극 활용하세요.
