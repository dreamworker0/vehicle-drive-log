# 글로벌 룰: PWA 및 오프라인 우선주의 체크 (Offline-First Validation)

이 규칙은 `차량운행일지` 프로젝트에서 데이터를 읽거나 쓰는 기능을 추가/수정할 때 **반드시 준수**해야 하는 글로벌 행동 규칙입니다.

## 1. 목적
차량운행일지 특성상 네트워크 환경이 열악한 장소(지하 주차장, 산간 외곽)에서 데이터 입력을 시도할 수 있습니다. 기능을 구현할 때 오프라인 상황과 PWA 특성을 고려한 코드를 우선적으로 작성하기 위함입니다.

## 2. 세부 원칙
- **선행 룰 확인**: 데이터 CUD(생성/수정/삭제) 로직 구현 요청을 받을 시 항상 이 규칙을 염두에 둡니다.
- **오프라인 감지**: 사용자 디바이스의 네트워크 상태(`navigator.onLine` 또는 관련된 hook)를 인지하고 UI에 반영되어야 합니다.
- **Firestore 캐싱 고려**: Firestore의 Offline Persistence 기능을 켜두었거나 관련 hook(`useRetry` 등)을 사용하는 경우 무한 로딩 상태에 빠지지 않도록 Timeout과 재시도(Retry) 로직을 결합해야 합니다.
- **낙관적 업데이트 (Optimistic Update)**: UI에서는 일단 업데이트된 화면을 먼저 표시하여 사용자 경험의 중단을 방지해야 합니다.

## 3. 에이전트 체크리스트
에이전트는 사용자가 "저장 기능 만들어줘", "버튼 눌렀을 때 API 호출되게 해줘" 등을 요청할 때 다음을 체크해야 합니다:
1. "오프라인 상태에서 이 기능을 사용할 경우 어떻게 UI를 처리할 것인가?"
2. 네트워크 오류, 지연(Timeout) 시 재시도 로직(`runWithRetry` 또는 커스텀 큐)을 사용하는가?
3. 작성된 코드가 브라우저의 오프라인 상태 또는 서비스 워커와 충돌하지 않는가?

요구사항이 분명치 않다면 사용자와 위 항목들을 먼저 논의하세요.
