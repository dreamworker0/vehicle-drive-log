# 📅 1인 관리자 전용 차량운행일지 안정 운영 계획표

> 1인 관리자가 일상 업무 중에서 **놓치기 쉽지만 서비스 중단을 막기 위해 반드시 점검해야 하는 보안/백업/인프라 필수 체크리스트**와 정기 일정표입니다.

---

## 🚨 1인 관리자가 가장 놓치기 쉬운 5대 치명적 리스크

1. **"백업이 되고 있겠지?" 방치형 리스크**
   - 백업 Functions가 정상 동작한다고 믿었으나, Cloud Storage 권한 이슈나 Firebase 내부 변경으로 인해 실제 백업 폴더 내 파일이 생성되지 않고 있는 경우.
   - **해결책**: 매주 1회 백업 폴더를 직접 눈으로 확인하고, 반기 1회 복구 시뮬레이션을 진행합니다.

2. **"결제 수단 및 API 키 만료" 서비스 중단 리스크**
   - TMap API, EmailJS, Gemini API, 도메인/호스팅 등의 결제 수단(신용카드)이 만료되거나 연간 갱신 시점을 놓쳐 하루아침에 서비스가 먹통이 되는 경우.
   - **해결책**: 관리자의 개인 캘린더(Google 캘린더 등)에 만료 30일 전 알림을 등록하고, 분기별로 결제 수단을 사전 점검합니다.

3. **"보안 취약점 방치" 해킹 및 데이터 유출 리스크**
   - npm 패키지 의존성에 등록된 취약점이 방치되거나 Firebase 보안 규칙(`firestore.rules`)의 틈새로 인해 악성 공격이나 데이터 유출이 발생하는 경우.
   - **해결책**: 월 1회 `/audit` 워크플로우를 실행하고 발견된 취약점을 패치합니다.

4. **"비용 과다 청구 및 할당량 초과" 리스크**
   - 크롤러 공격, 악성 요청, 혹은 코드상의 무한 루프 버그로 인해 Firestore/Functions 요청이 폭발하여 과도한 비용이 청구되거나 일일 무료 할당량이 조기 소진되는 경우.
   - **해결책**: GCP 결제 콘솔에서 예산 알림(Budget Alert)을 반드시 설정하고, 매달 사용량 트렌드를 모니터링합니다.

5. **"장애 대처 훈련 미흡" 패닉 리스크**
   - 배포 후 예기치 못한 치명적 버그가 발생했을 때 롤백 명령어가 기억나지 않아 우왕좌왕하며 복구 골든타임을 놓치는 경우.
   - **해결책**: 긴급 롤백 워크플로우(`/rollback`)를 사전에 숙지하고 분기별로 가상 시뮬레이션을 수행합니다.

## 📅 주기별 안정 운영 로드맵

### 1. 매일 (Daily - 약 3분 소요)
*아침 출근 후 커피 한 잔 마시며 가볍게 모니터링*
- [ ] **Functions 에러 여부 확인**:
  - `npm run health` 실행 또는 Firebase 콘솔 접속하여 최근 24시간 동안 에러 발생 건수 확인.
- [ ] **Sentry 알림 확인**:
  - 메일로 들어온 프론트엔드 실시간 에러 리포트가 있는지 확인.
- [ ] **어제 백업 정상 여부 확인**:
  - Cloud Storage 내 `backups/firestore/YYYY-MM-DD/` 폴더가 정상적으로 생성되었는지 확인.

### 2. 매주 (Weekly - 약 10분 소요)
*매주 월요일 오전 주간 루틴*
- [ ] **의존성 보안 감사**:
  - `/audit` 워크플로우를 실행해 치명적인 패키지 보안 취약점이 감지되었는지 점검.
- [ ] **비정상 트래픽 및 사용량 급증 체크**:
  - Firebase Console의 Firestore/Functions Usage 탭을 열어 비정상적인 읽기/쓰기 급증 패턴이 없는지 확인.

### 3. 매달 (Monthly - 약 20분 소요)
*매월 1일에 진행하는 정기 시스템 정화 및 보안 정비*
- [ ] **API 및 환경변수 상태 점검**:
  - `functions/.env`에 등록된 TMap, EmailJS, Gemini API 키의 사용량 한도 및 결제 수단 유효 기간 상태 확인.
- [ ] **시스템 클린업 실행**:
  - `/cleanup` 워크플로우를 실행하여 미사용 패키지, 캐시, 불필요한 임시 파일을 정리해 군더더기 없는 빌드 환경 유지.
- [ ] **Storage 수명 주기 상태 확인**:
  - `/storage-lifecycle` 설정에 따라 임시 폴더 내 오래된 데이터가 정상적으로 자동 영구 삭제되고 있는지 용량 확인.

### 4. 매 분기 (Quarterly - 약 30분 소요)
*3개월마다 진행하는 예방 정비*
- [ ] **비상 대응 및 롤백 리허설**:
  - `ROLLBACK.md` 문서를 정독하고, 로컬 에뮬레이터 환경에서 실제로 `/rollback` 프로세스를 작동해 봄으로써 비상상황 행동 요령 숙지.
- [ ] **도메인 및 SSL 인증서 만료일 점검**:
  - 도메인 구매 대행 사이트 및 인증서 상태를 확인하여 자동 연장 결제가 원활한지 확인.

### 5. 반기 (Half-Yearly - 약 1시간 소요)
*6개월마다 진행하는 중점 인프라 점검*
- [ ] **데이터 실제 복원 테스트 (DR Test)**:
  - 백업된 Firestore JSON 데이터의 일부를 로컬 에뮬레이터 환경에 임포트(`npm run db:seed` 응용)하여, 데이터 무결성이 깨지지 않고 제대로 정상 복구되는지 검증.
- [ ] **GCP / Firebase 액세스 권한 일제 점검**:
  - Firebase 프로젝트 협업자에 퇴사한 사람이나 불필요한 계정이 그대로 남아있는지 권한 권고 확인 및 정리.

## 🛠️ 1인 관리자를 위한 긴급 장애 대처 치트시트

장애 상황 시 당황하지 않고 터미널에서 실행할 3대 핵심 워크플로우 명령어입니다.

### ① 긴급 롤백 (최근 정상 배포 버전 복원)
서비스 배포 직후 치명적 오류가 발견되면 지체 없이 아래 명령어를 제안하고 실행합니다.
```bash
# 이전 정상 상태 빌드로 즉시 Hosting 및 Rules 원복
npm run rollback
```

### ② 긴급 시스템 상태 및 에러 로그 조회
서버가 죽거나 이상 동작할 때, 최근 50줄의 에러 로그와 헬스 체크 결과를 확인합니다.
```bash
# Cloud Functions 헬스체크 및 에러 감지
npm run health
```

### ③ 보안 취약점 즉시 진단 및 해결
보안 우려 사항이 발생했을 때 신속하게 진단을 수행합니다.
```bash
# 보안 진단 워크플로우 실행
npm run audit
```
