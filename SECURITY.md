# 보안 정책 (Security Policy)

차량 운행일지는 사회복지기관·비영리단체의 실제 운영 데이터를 다루는 멀티테넌트 서비스입니다. 보안 취약점 제보에 항상 감사드립니다.

## 취약점 제보 방법

> **공개 이슈(Issues)로 취약점을 신고하지 마세요.** 악용 가능한 정보가 노출됩니다.

취약점을 발견하면 아래로 **비공개** 제보해 주세요.

- 📧 이메일: **ehsheh@gmail.com**
- 또는 GitHub **Security Advisories**(저장소 → Security → Report a vulnerability)

제보 시 다음을 포함해 주시면 확인이 빠릅니다.

- 취약점 유형과 영향 범위 (예: 다른 기관 데이터 접근, 권한 상승, 인증 우회 등)
- 재현 절차 또는 PoC
- 관련 파일·엔드포인트 경로 (알고 있는 경우)

## 대응 절차

1. 영업일 기준 **3일 이내** 접수 확인 회신
2. 영향도 평가 후 수정 계획 공유
3. 패치 배포 후 제보자와 공개 시점 조율

확인된 취약점은 배포 파이프라인(CI → 프로덕션)을 통해 우선 처리됩니다.

## 범위

- **대상**: 이 저장소의 코드, Cloud Functions, Firestore/Storage 보안 규칙
- **대상 외**: 제3자 서비스(Firebase, Sentry, Gemini, 알리고 등) 자체의 취약점, 운영 인프라 설정 — 각 서비스 제공자에게 직접 제보해 주세요.

## 알아두면 좋은 것

- 클라이언트 번들에 포함된 Firebase 웹 설정값(`apiKey` 등)은 **공개되도록 설계된 값**이며 비밀이 아닙니다. 데이터 접근은 Firestore/Storage 보안 규칙과 App Check로 통제됩니다.
- 서버측 비밀정보(서비스 계정 키, Gemini API 키 등)는 저장소에 커밋되지 않습니다.
